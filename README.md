# 📥 Cross Platform DFIR

This module focuses on the forensic acquisition of email data from the Polish provider **WP.pl** using **Mozilla Thunderbird** as an intermediary tool for local evidence preservation on macOS.

### 🛠️ Forensic Toolkit
* **Client:** Mozilla Thunderbird (macOS build)
* **Protocols:** IMAP (Synchronization) / SMTP
* **Storage:** MBOX (Berkeley Mailbox) format

### 📁 Investigation Roadmap

Each directory contains a theoretical introduction, practical command-line execution, and a list of artifacts generated by these actions (to account for the forensic footprint).

1. [**Thunderbird Email Acquisition**](./1_Thunderbird_Email_Acquisition_&_WP.pl) – Forensic preservation of IMAP-based webmail (WP.pl) through local client synchronization, evidence validation, and MBOX artifact extraction.
   
### ⚠️ DFIR Best Practices
* **Forensic Footprint:** Be aware that Thunderbird modifies `global-messages-db.sqlite` and creates ULS logs during the synchronization process.
* **Integrity:** Always perform SHA-256 hashing of the `ImapMail` directory immediately after acquisition to maintain the chain of custody.
